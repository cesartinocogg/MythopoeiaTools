📖 FWF Studio · User Manual

FWF Studio is a complete web‑based worldbuilding suite. It combines a compendium of entries (characters, locations, items, lore), an interactive map editor with layers and linked pins/zones, a relation board to visualise connections, a timeline for historical events, and powerful export tools. Everything is stored in a single portable .fwf file.

This manual covers every feature, from the first launch to advanced workflows.

🏠 Home

Dashboard with world stats, cover, and quick access to all sections.

📄 Compendium

Create, edit, and browse entries with Markdown, tags, categories, and wiki links.

🗺️ Maps

Draw pins and zones on layered maps, link them to entries, export as PNG/JPG/SVG.

🕸️ Relations

Build an interactive graph of entities, add custom relation labels, drag nodes.

⏳ Timeline

Log historical milestones with years, titles, and rich descriptions (Markdown).

⚙️ World Config

Set world name, description, cover, background, and manage categories.

💡 Key concept: Everything is interconnected. Entries can be linked via @mentions (wiki style) in any text field. Pins/zones on maps can point to entries. The relation board visually maps connections. The timeline entries can also contain mentions.

All data – including images – is embedded inside the .fwf file. Save, load, share with a single file.

🖥️ Interface Overview

The interface is built around a fixed top bar and a main workspace that changes according to the selected tab.

Top Bar

  • Brand / World name – click to go to Home.
  • Navigation tabs – switch between main sections.
  • ⚙️ World & Categories – opens the configuration modal.
  • Status indicator – 🟢 Ready (no pending changes) or 🟡 Unsaved (changes made).
  • 📤 Export – opens export options (native .fwf, Markdown, JS module).
  • 📂 Load .fwf – load a previously saved world file.
  • 💾 Save .fwf – save current state to a compressed .fwf file.
  • 🌐 ES / EN – toggle language (persistent in browser).

Workspace

Each tab presents a different workspace:

  • Home – world summary, statistics, quick‑access cards.
  • Compendium – entry list with search/filters, plus viewer/editor.
  • Maps – interactive canvas with layers, tools, and properties panel.
  • Relations – draggable node graph with zoom/pan.
  • Timeline – vertical timeline with event cards.
🔄 Auto‑save indicator: Any modification marks the state as “dirty”. The status pill turns yellow. Don’t forget to save your work with 💾 Save .fwf before closing.

📄 Compendium

The compendium is the heart of FWF Studio – it stores all your world entries. Each entry has a title, category, tags, an optional cover image, and Markdown content.

Entry List

  • Search bar – filters by title or content.
  • Category filter – dropdown with all defined categories.
  • Tag filter – type a tag to show only entries containing it.
  • + New Entry – creates a new entry and opens the editor.

Each card shows: cover image (if any), category icon/name, title, a content snippet, and clickable tags. Click a tag to filter by it.

Viewer

Click any entry card to open the viewer. It displays:

  • Cover image, category, title.
  • Rendered Markdown content (with active wiki links).
  • Tags (clickable to filter).
  • Relations section – shows all links from/to this entry on the relation board. Click a relation to jump to the linked entry.

Buttons: ⬅ Back to Compendium, ✏️ Edit Entry, 🗑️ Delete.

Editor

The editor is split into two panels:

  • Left panel (edit): Title, Category, Tags (comma‑separated), 🖼️ Cover button to upload an image, and the main Markdown content area. The content field supports @mentions (see Advanced section).
  • Right panel (preview): live preview of the rendered Markdown, with wiki links active.

Buttons: ⬅ Back to Viewer (saves changes) and ✔ Save Changes (saves and stays in editor).

🗺️ Maps

The map editor supports multiple maps, each with its own background image, layers, and objects (pins and zones). Objects can be linked to compendium entries.

Map Management

  • Map selector – switch between existing maps.
  • + New Map – create a new map (name required).
  • 🗑️ Delete Map – remove current map (at least one map must remain).

Layers

  • Each map has layers (starting with a base layer). Layers help organise visual elements.
  • Active layer selector – new objects are added to the selected layer.
  • Layer list – each layer has a visibility checkbox and a delete button (except the last layer).
  • + New – add a new layer.

Tools

  • 📍 Pin – click to add a pin at the centre of the canvas (or current cursor position). Drag to reposition.
  • 🛑 Zone – activate polygon drawing mode. Click on the canvas to add vertices. Press Enter to close the polygon. The zone appears with a label on top.

When an object is selected, the Properties panel appears:

  • Name / Text – displayed label.
  • For zones: font size, fill colour, and opacity of the polygon (text remains fully opaque).
  • Link to entry – dropdown with all entries. If linked, the entry’s viewer will show a marker 📍 pointing to this map.
  • 🗑️ Remove – deletes the selected object.

Export Canvas

Buttons PNG, JPG, SVG download the current map view (including background and all visible layers).

🖱️ Navigation: Scroll to zoom, Alt+drag to pan.

🕸️ Relations

The relation board is a free‑form canvas where each node represents a compendium entry, and edges represent custom relations between them.

Adding Nodes

  • Use the dropdown in the top bar (only entries not already on the board are shown).
  • Click + Add to Board – the node appears at the centre of the visible area.

Interacting with Nodes

  • Drag – click and hold on any node to reposition it.
  • 📖 View – opens a modal with the entry’s full content (Markdown rendered, with tags and relations).
  • 🟢 Link – starts the linking mode. Click another node to open a dialog where you can define both directions of the relation (e.g., “is ally of” and “is allied with”).
  • ✖ Remove – deletes the node and all its incident edges from the board.

Managing Edges

Each edge is drawn as a line with an arrowhead. The label in the middle shows the forward relation. Click the label to open an edit modal where you can:

  • Change both direction labels.
  • Delete the edge entirely.

Export

Use the buttons PNG, JPG, SVG to capture the entire board (including all nodes and edges).

🖱️ Navigation: Scroll to zoom, Alt+drag to pan. Nodes are independent of the zoom/pan.

⏳ Timeline

The timeline displays a chronological list of historical milestones. Each event has a year/date (free text), a title, and a description that supports Markdown and wiki links.

  • + New Historical Milestone – opens a modal to fill in the event details.
  • Events are shown in alternating cards (left/right) with the year, title, and rendered description.
  • Each card has ✏️ Edit and 🗑️ Delete buttons.

The timeline updates in real time and the counter on the Home page reflects the number of events.

⚙️ Advanced Features

World Configuration

Access via the ⚙️ World & Categories button in the top bar. Here you can:

  • Change the world name and description.
  • Set a cover image (displayed on the Home page).
  • Choose a background theme (presets or custom upload) and adjust its opacity.
  • Manage categories – add, delete, reorder icons, names, and colours. At least one category must exist.

Wiki Links (@mentions)

Any Markdown text field (entry content, timeline description) supports automatic linking to entries using the @ syntax:

  • @EntryTitle – links to the entry with that title (case‑insensitive).
  • @[entryId] – explicit link by ID (more robust).

While typing @ in the entry editor, a dropdown appears with matching entries. Select one to insert a proper @[id] link.

In the rendered view, existing links are shown in green (with a 📍 icon if the entry is also on a map). Broken links appear in orange and can be clicked to create a new entry with the suggested title.

The system automatically normalizes mentions: when you save an entry, all @Title references are converted to @[id] for stability.

.fwf File Format

The native file format is a JSON object compressed with LZString and stored with the .fwf extension. It includes:

  • World metadata (name, description, cover, background, opacity).
  • All categories.
  • All entries (with images as base64).
  • All maps (with background images, layers, objects – pins/zones – and their links).
  • Relation board nodes and edges.
  • Timeline events.

Because images are embedded, the file can be large, but it is completely self‑contained.

Export Options

From the 📤 Export panel (or the top bar button) you can:

  • Save Native File (.fwf) – same as the main save button.
  • Download as Markdown (.md) – generates a single Markdown file with the world name, description, and all entries (with content, category, and tags).
  • Download as JavaScript Module (.js) – exports the entire data object as FWF_WORLD_DATA (or as a CommonJS/ES module). Useful for integration with other tools.

Language Switching

The interface can be toggled between English and Spanish using the 🌐 ES/EN button. The choice is saved in your browser’s local storage.

Keyboard Shortcuts & Tips

  • Alt + drag on maps or relations board – pan.
  • Scroll – zoom on maps and relations board.
  • Enter – while drawing a zone on the map, closes the polygon.
  • In the entry editor, the @ autocomplete appears automatically; use arrow keys and Enter to select.
  • Click on any tag in the compendium to immediately filter the list by that tag.

📖 FWF Studio · Manual de Usuario

FWF Studio es una suite completa de creación de mundos basada en web. Combina un compendio de entradas (personajes, lugares, objetos, mitos), un editor de mapas interactivo con capas y pines/zonas vinculados, un tablero de relaciones para visualizar conexiones, una línea de tiempo para eventos históricos y potentes herramientas de exportación. Todo se almacena en un único archivo .fwf portátil.

Este manual cubre cada funcionalidad, desde el primer inicio hasta los flujos de trabajo avanzados.

🏠 Inicio

Panel con estadísticas del mundo, portada y accesos rápidos a todas las secciones.

📄 Compendio

Crea, edita y navega por entradas con Markdown, etiquetas, categorías y enlaces wiki.

🗺️ Mapas

Dibuja pines y zonas en mapas con capas, vincúlalos a entradas, exporta a PNG/JPG/SVG.

🕸️ Relaciones

Construye un grafo interactivo de entidades, añade etiquetas personalizadas y arrastra nodos.

⏳ Línea de Tiempo

Registra hitos históricos con año, título y descripciones enriquecidas (Markdown).

⚙️ Configuración

Define el nombre, descripción, portada, fondo del mundo y gestiona las categorías.

💡 Concepto clave: Todo está interconectado. Las entradas pueden enlazarse mediante @menciones (estilo wiki) en cualquier campo de texto. Los pines/zonas en los mapas pueden apuntar a entradas. El tablero de relaciones mapea visualmente las conexiones. Los eventos de la línea de tiempo también pueden contener menciones.

Todos los datos – incluidas las imágenes – se incrustan dentro del archivo .fwf. Guarda, carga y comparte con un solo archivo.

🖥️ Visión general de la interfaz

La interfaz se compone de una barra superior fija y un área de trabajo principal que cambia según la pestaña seleccionada.

Barra superior

  • Marca / Nombre del mundo – haz clic para ir a Inicio.
  • Pestañas de navegación – cambian entre las secciones principales.
  • ⚙️ Mundo y Categorías – abre el modal de configuración.
  • Indicador de estado – 🟢 Listo (sin cambios pendientes) o 🟡 Sin guardar (hay modificaciones).
  • 📤 Exportar – abre opciones de exportación (.fwf nativo, Markdown, módulo JS).
  • 📂 Cargar .fwf – carga un archivo de mundo previamente guardado.
  • 💾 Guardar .fwf – guarda el estado actual en un archivo .fwf comprimido.
  • 🌐 ES / EN – cambia el idioma (persistente en el navegador).

Área de trabajo

Cada pestaña presenta un espacio de trabajo diferente:

  • Inicio – resumen del mundo, estadísticas y tarjetas de acceso rápido.
  • Compendio – lista de entradas con buscador/filtros, además de visor/editor.
  • Mapas – lienzo interactivo con capas, herramientas y panel de propiedades.
  • Relaciones – grafo de nodos arrastrables con zoom/pan.
  • Línea de Tiempo – línea vertical con tarjetas de eventos.
🔄 Guardado automático (indicador): Cualquier modificación marca el estado como “sucio”. La pastilla de estado se vuelve amarilla. No olvides guardar tu trabajo con 💾 Guardar .fwf antes de cerrar.

📄 Compendio

El compendio es el corazón de FWF Studio: almacena todas las entradas de tu mundo. Cada entrada tiene título, categoría, etiquetas, una imagen de portada opcional y contenido en Markdown.

Lista de entradas

  • Barra de búsqueda – filtra por título o contenido.
  • Filtro por categoría – desplegable con todas las categorías definidas.
  • Filtro por etiqueta – escribe una etiqueta para mostrar solo las entradas que la contienen.
  • + Nueva Entrada – crea una nueva entrada y abre el editor.

Cada tarjeta muestra: imagen de portada (si existe), icono/nombre de categoría, título, un extracto del contenido y etiquetas cliqueables. Haz clic en una etiqueta para filtrar por ella.

Visor

Haz clic en cualquier tarjeta para abrir el visor. Muestra:

  • Imagen de portada, categoría y título.
  • Contenido Markdown renderizado (con enlaces wiki activos).
  • Etiquetas (cliqueables para filtrar).
  • Relaciones – sección que muestra todos los enlaces desde/hacia esta entrada en el tablero de relaciones. Haz clic en una relación para saltar a la entrada vinculada.

Botones: ⬅ Volver al Compendio, ✏️ Editar Entrada, 🗑️ Eliminar.

Editor

El editor se divide en dos paneles:

  • Panel izquierdo (edición): Título, Categoría, Etiquetas (separadas por comas), botón 🖼️ Portada para subir una imagen, y el área de contenido Markdown. El campo de contenido admite @menciones (ver sección Avanzado).
  • Panel derecho (vista previa): previsualización en vivo del contenido renderizado, con los enlaces wiki activos.

Botones: ⬅ Volver al Visor (guarda los cambios) y ✔ Guardar Cambios (guarda y permanece en el editor).

🗺️ Mapas

El editor de mapas admite múltiples mapas, cada uno con su propia imagen de fondo, capas y objetos (pines y zonas). Los objetos pueden vincularse a entradas del compendio.

Gestión de mapas

  • Selector de mapa – cambia entre los mapas existentes.
  • + Nuevo Mapa – crea un nuevo mapa (se requiere nombre).
  • 🗑️ Eliminar Mapa – elimina el mapa actual (debe quedar al menos uno).

Capas

  • Cada mapa tiene capas (comienza con una capa base). Las capas ayudan a organizar los elementos visuales.
  • Selector de capa activa – los objetos nuevos se añaden a la capa seleccionada.
  • Lista de capas – cada capa tiene una casilla de visibilidad y un botón de eliminar (excepto la última).
  • + Nueva – añade una capa.

Herramientas

  • 📍 Pin – haz clic para añadir un pin en el centro del lienzo (o en la posición actual del cursor). Arrástralo para reposicionarlo.
  • 🛑 Zona – activa el modo de dibujo de polígonos. Haz clic en el lienzo para añadir vértices. Presiona Enter para cerrar el polígono. La zona aparece con una etiqueta superpuesta.

Cuando se selecciona un objeto, aparece el panel de Propiedades:

  • Nombre / Texto – etiqueta mostrada.
  • Para zonas: tamaño de fuente, color de relleno y opacidad del polígono (el texto permanece totalmente opaco).
  • Vincular a entrada – desplegable con todas las entradas. Si se vincula, el visor de esa entrada mostrará un marcador 📍 apuntando a este mapa.
  • 🗑️ Quitar – elimina el objeto seleccionado.

Exportar lienzo

Los botones PNG, JPG, SVG descargan la vista actual del mapa (incluyendo fondo y todas las capas visibles).

🖱️ Navegación: Rueda para hacer zoom, Alt+arrastrar para desplazar el lienzo.

🕸️ Relaciones

El tablero de relaciones es un lienzo de forma libre donde cada nodo representa una entrada del compendio, y las aristas representan relaciones personalizadas entre ellas.

Añadir nodos

  • Usa el desplegable en la barra superior (solo se muestran las entradas que aún no están en el tablero).
  • Haz clic en + Añadir al Tablero – el nodo aparece en el centro del área visible.

Interactuar con los nodos

  • Arrastrar – haz clic y mantén sobre cualquier nodo para reposicionarlo.
  • 📖 Ver – abre un modal con el contenido completo de la entrada (Markdown renderizado, con etiquetas y relaciones).
  • 🟢 Vincular – inicia el modo de enlace. Haz clic en otro nodo para abrir un diálogo donde podrás definir ambas direcciones de la relación (ej. “es aliado de” y “es aliado con”).
  • ✖ Quitar – elimina el nodo y todas sus aristas incidentes del tablero.

Gestionar aristas

Cada arista se dibuja como una línea con punta de flecha. La etiqueta en el punto medio muestra la relación directa. Haz clic en la etiqueta para abrir un modal de edición donde puedes:

  • Cambiar ambas etiquetas de dirección.
  • Eliminar la arista por completo.

Exportar

Usa los botones PNG, JPG, SVG para capturar todo el tablero (incluyendo todos los nodos y aristas).

🖱️ Navegación: Rueda para hacer zoom, Alt+arrastrar para desplazar. Los nodos son independientes del zoom/pan.

⏳ Línea de Tiempo

La línea de tiempo muestra una lista cronológica de hitos históricos. Cada evento tiene un año/fecha (texto libre), un título y una descripción que admite Markdown y enlaces wiki.

  • + Nuevo Hito Histórico – abre un modal para completar los detalles del evento.
  • Los eventos se muestran en tarjetas alternadas (izquierda/derecha) con el año, el título y la descripción renderizada.
  • Cada tarjeta tiene botones ✏️ Editar y 🗑️ Eliminar.

La línea de tiempo se actualiza en tiempo real y el contador en la página de inicio refleja el número de eventos.

⚙️ Funciones avanzadas

Configuración del mundo

Accede mediante el botón ⚙️ Mundo y Categorías en la barra superior. Aquí puedes:

  • Cambiar el nombre del mundo y la descripción.
  • Establecer una imagen de portada (se muestra en la página de Inicio).
  • Elegir un tema de fondo (predefinidos o subida personalizada) y ajustar su opacidad.
  • Gestionar las categorías – añadir, eliminar, reordenar iconos, nombres y colores. Debe existir al menos una categoría.

Enlaces wiki (@menciones)

Cualquier campo de texto en Markdown (contenido de entradas, descripción de la línea de tiempo) admite enlaces automáticos a entradas mediante la sintaxis @:

  • @TítuloEntrada – enlaza a la entrada con ese título (no distingue mayúsculas).
  • @[idEntrada] – enlace explícito por ID (más robusto).

Mientras escribes @ en el editor de entradas, aparece un menú desplegable con las entradas coincidentes. Selecciona una para insertar un enlace @[id] correcto.

En la vista renderizada, los enlaces existentes se muestran en verde (con un icono 📍 si la entrada también está en un mapa). Los enlaces rotos aparecen en naranja y se pueden hacer clic para crear una nueva entrada con el título sugerido.

El sistema normaliza automáticamente las menciones: al guardar una entrada, todas las referencias @Título se convierten a @[id] para mayor estabilidad.

Formato de archivo .fwf

El formato nativo es un objeto JSON comprimido con LZString y almacenado con extensión .fwf. Incluye:

  • Metadatos del mundo (nombre, descripción, portada, fondo, opacidad).
  • Todas las categorías.
  • Todas las entradas (con imágenes en base64).
  • Todos los mapas (con imágenes de fondo, capas, objetos – pines/zonas – y sus vínculos).
  • Nodos y aristas del tablero de relaciones.
  • Eventos de la línea de tiempo.

Dado que las imágenes están incrustadas, el archivo puede ser grande, pero es completamente autónomo.

Opciones de exportación

Desde el panel 📤 Exportar (o el botón de la barra superior) puedes:

  • Guardar archivo nativo (.fwf) – igual que el botón principal de guardado.
  • Descargar como Markdown (.md) – genera un único archivo Markdown con el nombre del mundo, descripción y todas las entradas (con contenido, categoría y etiquetas).
  • Descargar como módulo JavaScript (.js) – exporta todo el objeto de datos como FWF_WORLD_DATA (o como módulo CommonJS/ES). Útil para integración con otras herramientas.

Cambio de idioma

La interfaz se puede alternar entre español e inglés mediante el botón 🌐 ES/EN. La elección se guarda en el almacenamiento local del navegador.

Atajos de teclado y consejos

  • Alt + arrastrar en mapas o en el tablero de relaciones – desplazar.
  • Rueda – zoom en mapas y tablero de relaciones.
  • Enter – mientras dibujas una zona en el mapa, cierra el polígono.
  • En el editor de entradas, el autocompletado de @ aparece automáticamente; usa las flechas y Enter para seleccionar.
  • Haz clic en cualquier etiqueta del compendio para filtrar la lista por esa etiqueta.
