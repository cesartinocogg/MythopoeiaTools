✦ Mythopoeia Maker · User Manual

Mythopoeia Maker is a visual storytelling tool designed for writers, game masters, and creators of interactive fiction. It combines a node-based passage editor with a freeform visual board, allowing you to map out branching narratives, connect ideas with decisions, and export your work as a playable visual novel, Markdown, or diagram.

Everything is stored in a portable .fmm file, making it easy to save, load, and share your story projects.

📄 Passages

Create story nodes with titles, emojis, colors, and rich Markdown content. Each passage is a scene or chapter.

🔗 Decisions

Add choices that link passages together. Build branching narratives with visual connections on the board.

🗺️ Visual Board

Drag and arrange passages on an infinite canvas. See the story structure at a glance with connection lines.

✨ Stickers & Emojis

Decorate your board with draggable emojis and icons. Visualize themes, characters, or locations.

🎮 Playable Export

Export your story as a standalone HTML visual novel. Test and share your narrative with one click.

💾 Native Format

Save your project as a compressed .fmm file. All passages, decisions, stickers, and layout are preserved.

💡 Key concept: Each passage is a scene. Decisions create branching paths. The board visualizes the entire story graph. Use @mentions to link passages directly from the editor, and the board connections update automatically.

All data – including emojis and layout – is embedded in the project file. Save, load, and share with a single .fmm file.

🖥️ Interface Overview

The interface is divided into three main areas: the header (top), the sidebar (left), and the board (main view). The sidebar can be collapsed with the ☰ button.

Header

  • ☰ Toggle Sidebar – show/hide the passage list.
  • ✦ MYTHOPOEIA – the brand and title.
  • ▶️ Test Play – opens a popup with the playable visual novel version of your story.
  • ➕ New Passage – creates a new passage and opens the editor panel.
  • 📜 Export ▾ – dropdown with export options: Playable HTML, Markdown, Mermaid Diagram, PNG Image.
  • ✨ Icons & Emojis – toggles the emoji palette drawer for adding stickers to the board.
  • 💾 Save .FMM – downloads your project as a compressed .fmm file.
  • 📂 Load .FMM – opens a file picker to load a previously saved project.

Sidebar

  • 📖 Passages – list of all passages in your project.
  • Count badge – shows the total number of passages.
  • Passage items – click to select and center a passage on the board. Double-click to open it in the editor.
  • The ✦ icon marks the starting passage (first in the list).

Board

  • The infinite canvas where passages and stickers are placed.
  • A subtle grid helps with visual alignment.
  • Passage nodes show the title, emoji, body preview, and decisions.
  • Connection lines (curves with arrows) link passages via decisions.
  • Zoom & Pan: scroll to zoom, drag the background to pan.
  • HUD: bottom-right shows zoom level and a 🎯 Center Board button.
🖱️ Navigation: Scroll to zoom • Drag the background to pan • Click a passage to select it • Double-click a passage to edit • Drag passages to reposition them.

📄 Passages

Passages are the building blocks of your story. Each passage represents a scene, chapter, or narrative node.

Creating a Passage

  • Click ➕ New Passage in the header. A new passage is created and the editor panel opens.
  • Alternatively, use the dropdown in the decisions editor to create a new target passage when linking.

Editor Panel

The editor panel slides in from the right when you create or double-click a passage.

  • Emoji selector – choose an icon for the passage (displayed on the node).
  • Color picker – sets the border color of the node.
  • Title input – the passage title. The slug (URL-friendly ID) is auto-generated.
  • Content editor – write your story in Markdown. Supports @mentions for linking.
  • Decisions editor – add, edit, or delete choices. Each decision links to another passage.
  • Live Preview – shows rendered Markdown with active links to other passages.

Markdown & @Mentions

The content editor supports standard Markdown:

  • **bold** → bold
  • *italic* → italic
  • @slug – links to another passage by its slug. The slug is auto-generated from the title.
  • While typing @, an autocomplete dropdown appears with matching passages.
  • In the preview, linked passages appear as clickable gold badges.
💡 Pro tip: Use @mentions to create hyperlinks between passages directly from the editor. The board connections update automatically when you save.

Saving & Deleting

  • 💾 Save – saves the current passage and closes the panel.
  • 🗑️ Delete – removes the passage. All decisions pointing to it are cleaned up.
  • 🎯 Center on Board – focuses the board on the current passage.

🔗 Decisions & Connections

Decisions create the branching paths of your narrative. Each decision is a choice that leads to another passage.

Managing Decisions

In the editor panel, the Decisions / Choices section lets you:

  • + Add Decision – adds a new choice row.
  • Decision text – the text displayed to the player (e.g., "Enter the castle").
  • Target passage – select a passage from the dropdown. Choose ➕ Create New... to make a new passage on the fly.
  • ✕ Delete – removes a decision row.

Visual Connections

  • On the board, decisions are shown as curved lines with arrows pointing to the target passage.
  • Each decision appears as a small item inside the source passage node.
  • Hovering over a connection line highlights it in gold.
  • When you move a passage, all its connections update automatically.

Starting Passage

The first passage in the list is marked with a ✦ Starting Passage badge. This is where the visual novel begins. You can reorder passages by editing the .fmm file, but the tool currently uses the first entry as the start.

🔗 Link integrity: If you delete a passage, all decisions that pointed to it are automatically removed from other passages. No broken links remain.

🗺️ Board Navigation & Customization

The board is your story's visual map. It displays all passages as movable nodes with their connections.

Navigation

  • Zoom – scroll up/down to zoom in/out. Zoom level is displayed in the bottom-right corner.
  • Pan – click and drag the empty background to pan around the infinite canvas.
  • Center Board – click the 🎯 Center Board button to auto-fit all passages into view.
  • Center on Passage – click a passage in the sidebar to center it on the board.

Interacting with Passages

  • Click – selects a passage (highlighted with a gold border).
  • Double-click – opens the passage in the editor panel.
  • Drag – move a passage anywhere on the board. All connections update in real time.
  • Passage nodes show: emoji, title, body preview (truncated), and a list of decisions.

Stickers & Emojis

  • Click ✨ Icons & Emojis in the header to open the palette drawer.
  • Drag any emoji from the palette onto the board to place it as a sticker.
  • Stickers are free-floating and can be dragged anywhere. They are saved with the project.
  • Use stickers to mark locations, characters, themes, or add visual flair to your board.
🎯 Quick tip: Double-click any passage in the sidebar to instantly open it for editing. Click a passage on the board to select it, then use the sidebar entry list for centering.

📤 Export & Project Management

Export Options

Access all exports from the 📜 Export ▾ dropdown in the header.

  • 🎮 Playable Game (.HTML) – generates a standalone HTML visual novel. Open in any browser, click decisions to progress. All passages, decisions, and formatting are preserved.
  • 📝 Markdown (.MD) – exports the entire story as a single Markdown file with all passages, their content, and decision links (as anchors).
  • 🧜‍♂️ Mermaid Diagram – generates a Mermaid.js graph definition. Paste it into any Mermaid-compatible viewer to see the story structure.
  • 🖼️ PNG Image – captures the current board view as a PNG image. Includes all passages, stickers, and connection lines.

Test Play

The ▶️ Test Play button opens a popup window with the playable visual novel version of your story. This is a quick way to playtest your narrative without exporting. The popup uses the same engine as the HTML export.

Project File (.fmm)

The native format is a JSON file with the .fmm extension. It stores:

  • All passages (id, title, slug, emoji, color, body, position, decisions).
  • All stickers (id, emoji, position).
  • Layout information (passage positions).

Because everything is stored in plain JSON, it's easy to version control or edit manually.

Save & Load

  • 💾 Save .FMM – downloads the current project as a .fmm file.
  • 📂 Load .FMM – opens a file picker to load a previously saved project. The current project is replaced.

Data is also saved automatically to your browser's localStorage between sessions. Your work persists even if you close the tab.

🔄 Auto-save: Your project is automatically saved to localStorage. The .fmm export is a portable backup and sharing format.

⚙️ Advanced Features

@Mention Autocomplete

The content editor includes an intelligent autocomplete system:

  • Type @ followed by a few letters to see matching passages.
  • Use ↑ / ↓ to navigate the list.
  • Press Enter or Tab to insert the selected mention.
  • Press Escape to close the dropdown.
  • The link is inserted as @slug, which renders as a clickable link in the preview.

Keyboard Shortcuts

  • Scroll – zoom on the board.
  • Drag background – pan the board.
  • Double-click passage – open editor.
  • Enter (in editor) – when typing a mention, selects it.
  • Escape – closes the mention dropdown or the editor panel.

Stickers & Visual Mapping

  • Stickers are independent of passages and can be used to mark regions, themes, characters, or locations.
  • They are stored with their position and persist across saves.
  • Drag stickers freely – they don't snap to any grid.

Story Structure Tips

  • Keep your starting passage simple – it sets the tone for the entire experience.
  • Use decisions to create meaningful choices, not just "continue" buttons.
  • Use @mentions to create hyperlinks within text for non-linear reading.
  • Use the board to spot disconnected passages or bottlenecks in your narrative.
  • Test Play often – it's the fastest way to experience your story from a reader's perspective.
🧪 Experimental: The .fmm format is plain JSON. You can edit it manually to reorder passages (the first is the start) or adjust positions with precision.

✦ Mythopoeia Maker · Manual de Usuario

Mythopoeia Maker es una herramienta de narración visual diseñada para escritores, directores de juego y creadores de ficción interactiva. Combina un editor de pasajes basado en nodos con un tablero visual de forma libre, permitiéndote mapear narrativas ramificadas, conectar ideas con decisiones y exportar tu trabajo como una novela visual jugable, Markdown o diagrama.

Todo se almacena en un archivo .fmm portátil, lo que facilita guardar, cargar y compartir tus proyectos narrativos.

📄 Pasajes

Crea nodos de historia con títulos, emojis, colores y contenido Markdown enriquecido. Cada pasaje es una escena o capítulo.

🔗 Decisiones

Añade opciones que enlazan pasajes. Construye narrativas ramificadas con conexiones visuales en el tablero.

🗺️ Tablero Visual

Arrastra y organiza pasajes en un lienzo infinito. Visualiza la estructura de la historia de un vistazo con líneas de conexión.

✨ Pegatinas & Emojis

Decora tu tablero con emojis e iconos arrastrables. Visualiza temas, personajes o ubicaciones.

🎮 Exportación Jugable

Exporta tu historia como una novela visual HTML independiente. Prueba y comparte tu narrativa con un clic.

💾 Formato Nativo

Guarda tu proyecto como un archivo .fmm comprimido. Todos los pasajes, decisiones, pegatinas y diseño se conservan.

💡 Concepto clave: Cada pasaje es una escena. Las decisiones crean caminos ramificados. El tablero visualiza todo el grafo de la historia. Usa @menciones para enlazar pasajes directamente desde el editor, y las conexiones del tablero se actualizan automáticamente.

Todos los datos – incluidos emojis y diseño – están incrustados en el archivo del proyecto. Guarda, carga y comparte con un solo archivo .fmm.

🖥️ Visión general de la interfaz

La interfaz se divide en tres áreas principales: el encabezado (arriba), la barra lateral (izquierda) y el tablero (vista principal). La barra lateral se puede ocultar con el botón ☰.

Encabezado

  • ☰ Alternar barra lateral – muestra/oculta la lista de pasajes.
  • ✦ MYTHOPOEIA – la marca y el título.
  • ▶️ Probar – abre una ventana emergente con la versión jugable de tu historia.
  • ➕ Nuevo Pasaje – crea un nuevo pasaje y abre el panel del editor.
  • 📜 Exportar ▾ – menú desplegable con opciones: HTML jugable, Markdown, Diagrama Mermaid, PNG.
  • ✨ Iconos & Emojis – alterna el cajón de emojis para añadir pegatinas al tablero.
  • 💾 Guardar .FMM – descarga tu proyecto como un archivo .fmm comprimido.
  • 📂 Cargar .FMM – abre un selector de archivos para cargar un proyecto guardado.

Barra lateral

  • 📖 Pasajes – lista de todos los pasajes de tu proyecto.
  • Contador – muestra el número total de pasajes.
  • Elementos de pasaje – haz clic para seleccionar y centrar un pasaje en el tablero. Haz doble clic para abrirlo en el editor.
  • El icono ✦ marca el pasaje de inicio (el primero de la lista).

Tablero

  • El lienzo infinito donde se colocan pasajes y pegatinas.
  • Una cuadrícula sutil ayuda con la alineación visual.
  • Los nodos de pasaje muestran el título, emoji, vista previa del cuerpo y las decisiones.
  • Las líneas de conexión (curvas con flechas) enlazan pasajes a través de decisiones.
  • Zoom & Panorámica: desplázate para hacer zoom, arrastra el fondo para moverte.
  • HUD: en la esquina inferior derecha muestra el nivel de zoom y un botón 🎯 Centrar tablero.
🖱️ Navegación: Desplázate para hacer zoom • Arrastra el fondo para moverte • Haz clic en un pasaje para seleccionarlo • Haz doble clic en un pasaje para editarlo • Arrastra los pasajes para reposicionarlos.

📄 Pasajes

Los pasajes son los bloques de construcción de tu historia. Cada pasaje representa una escena, capítulo o nodo narrativo.

Crear un pasaje

  • Haz clic en ➕ Nuevo Pasaje en el encabezado. Se crea un nuevo pasaje y se abre el panel del editor.
  • Alternativamente, usa el menú desplegable en el editor de decisiones para crear un nuevo pasaje de destino al enlazar.

Panel del editor

El panel del editor se desliza desde la derecha cuando creas o haces doble clic en un pasaje.

  • Selector de emoji – elige un icono para el pasaje (se muestra en el nodo).
  • Selector de color – establece el color del borde del nodo.
  • Campo de título – el título del pasaje. El slug (ID amigable para URL) se genera automáticamente.
  • Editor de contenido – escribe tu historia en Markdown. Admite @menciones para enlazar.
  • Editor de decisiones – añade, edita o elimina opciones. Cada decisión enlaza a otro pasaje.
  • Vista previa en vivo – muestra el Markdown renderizado con enlaces activos a otros pasajes.

Markdown & @Menciones

El editor de contenido admite Markdown estándar:

  • **negrita** → negrita
  • *cursiva* → cursiva
  • @slug – enlaza a otro pasaje por su slug. El slug se genera automáticamente a partir del título.
  • Mientras escribes @, aparece un menú desplegable con los pasajes coincidentes.
  • En la vista previa, los pasajes enlazados aparecen como insignias doradas cliqueables.
💡 Consejo profesional: Usa @menciones para crear hipervínculos entre pasajes directamente desde el editor. Las conexiones del tablero se actualizan automáticamente al guardar.

Guardar y eliminar

  • 💾 Guardar – guarda el pasaje actual y cierra el panel.
  • 🗑️ Eliminar – elimina el pasaje. Todas las decisiones que apuntaban a él se limpian.
  • 🎯 Centrar en tablero – enfoca el tablero en el pasaje actual.

🔗 Decisiones & Conexiones

Las decisiones crean los caminos ramificados de tu narrativa. Cada decisión es una opción que lleva a otro pasaje.

Gestionar decisiones

En el panel del editor, la sección Decisiones / Opciones te permite:

  • + Añadir Decisión – añade una nueva fila de opción.
  • Texto de decisión – el texto que se muestra al jugador (ej. "Entrar al castillo").
  • Pasaje de destino – selecciona un pasaje del desplegable. Elige ➕ Crear Nuevo... para crear un nuevo pasaje sobre la marcha.
  • ✕ Eliminar – elimina una fila de decisión.

Conexiones visuales

  • En el tablero, las decisiones se muestran como líneas curvas con flechas que apuntan al pasaje de destino.
  • Cada decisión aparece como un elemento pequeño dentro del nodo del pasaje origen.
  • Al pasar el cursor sobre una línea de conexión, se resalta en dorado.
  • Cuando mueves un pasaje, todas sus conexiones se actualizan automáticamente.

Pasaje de inicio

El primer pasaje de la lista está marcado con una insignia ✦ Pasaje de inicio. Es donde comienza la novela visual. Puedes reordenar los pasajes editando el archivo .fmm, pero la herramienta utiliza la primera entrada como inicio.

🔗 Integridad de enlaces: Si eliminas un pasaje, todas las decisiones que apuntaban a él se eliminan automáticamente de otros pasajes. No quedan enlaces rotos.

🗺️ Navegación y personalización del tablero

El tablero es el mapa visual de tu historia. Muestra todos los pasajes como nodos móviles con sus conexiones.

Navegación

  • Zoom – desplázate hacia arriba/abajo para acercar/alejar. El nivel de zoom se muestra en la esquina inferior derecha.
  • Panorámica – haz clic y arrastra el fondo vacío para moverte por el lienzo infinito.
  • Centrar tablero – haz clic en el botón 🎯 Centrar tablero para ajustar automáticamente todos los pasajes a la vista.
  • Centrar en pasaje – haz clic en un pasaje en la barra lateral para centrarlo en el tablero.

Interactuar con los pasajes

  • Clic – selecciona un pasaje (resaltado con un borde dorado).
  • Doble clic – abre el pasaje en el panel del editor.
  • Arrastrar – mueve un pasaje a cualquier lugar del tablero. Todas las conexiones se actualizan en tiempo real.
  • Los nodos de pasaje muestran: emoji, título, vista previa del cuerpo (truncada) y una lista de decisiones.

Pegatinas & Emojis

  • Haz clic en ✨ Iconos & Emojis en el encabezado para abrir el cajón de la paleta.
  • Arrastra cualquier emoji de la paleta al tablero para colocarlo como pegatina.
  • Las pegatinas flotan libremente y se pueden arrastrar a cualquier lugar. Se guardan con el proyecto.
  • Usa pegatinas para marcar ubicaciones, personajes, temas o añadir estilo visual a tu tablero.
🎯 Consejo rápido: Haz doble clic en cualquier pasaje de la barra lateral para abrirlo al instante para editar. Haz clic en un pasaje del tablero para seleccionarlo y luego usa la lista de la barra lateral para centrarlo.

📤 Exportación y gestión de proyectos

Opciones de exportación

Accede a todas las exportaciones desde el menú desplegable 📜 Exportar ▾ en el encabezado.

  • 🎮 Juego jugable (.HTML) – genera una novela visual HTML independiente. Abre en cualquier navegador, haz clic en las decisiones para avanzar. Todos los pasajes, decisiones y formato se conservan.
  • 📝 Markdown (.MD) – exporta toda la historia como un único archivo Markdown con todos los pasajes, su contenido y enlaces de decisiones (como anclas).
  • 🧜‍♂️ Diagrama Mermaid – genera una definición de gráfico Mermaid.js. Pégalo en cualquier visor compatible con Mermaid para ver la estructura de la historia.
  • 🖼️ Imagen PNG – captura la vista actual del tablero como una imagen PNG. Incluye todos los pasajes, pegatinas y líneas de conexión.

Probar

El botón ▶️ Probar abre una ventana emergente con la versión jugable de tu historia. Es una forma rápida de probar tu narrativa sin exportar. La ventana emergente utiliza el mismo motor que la exportación HTML.

Archivo de proyecto (.fmm)

El formato nativo es un archivo JSON con extensión .fmm. Almacena:

  • Todos los pasajes (id, título, slug, emoji, color, cuerpo, posición, decisiones).
  • Todas las pegatinas (id, emoji, posición).
  • Información de diseño (posiciones de los pasajes).

Como todo se almacena en JSON plano, es fácil controlar versiones o editar manualmente.

Guardar y cargar

  • 💾 Guardar .FMM – descarga el proyecto actual como un archivo .fmm.
  • 📂 Cargar .FMM – abre un selector de archivos para cargar un proyecto guardado. El proyecto actual se reemplaza.

Los datos también se guardan automáticamente en el localStorage del navegador entre sesiones. Tu trabajo persiste incluso si cierras la pestaña.

🔄 Guardado automático: Tu proyecto se guarda automáticamente en localStorage. La exportación .fmm es una copia de seguridad portátil y un formato para compartir.

⚙️ Funciones avanzadas

Autocompletado de @menciones

El editor de contenido incluye un sistema de autocompletado inteligente:

  • Escribe @ seguido de unas pocas letras para ver los pasajes coincidentes.
  • Usa ↑ / ↓ para navegar por la lista.
  • Presiona Enter o Tab para insertar la mención seleccionada.
  • Presiona Escape para cerrar el menú desplegable.
  • El enlace se inserta como @slug, que se renderiza como un enlace cliqueable en la vista previa.

Atajos de teclado

  • Scroll – zoom en el tablero.
  • Arrastrar fondo – mover el tablero.
  • Doble clic en pasaje – abrir editor.
  • Enter (en el editor) – al escribir una mención, la selecciona.
  • Escape – cierra el menú desplegable de menciones o el panel del editor.

Pegatinas & mapeo visual

  • Las pegatinas son independientes de los pasajes y se pueden usar para marcar regiones, temas, personajes o ubicaciones.
  • Se almacenan con su posición y persisten entre guardados.
  • Arrastra las pegatinas libremente – no se ajustan a ninguna cuadrícula.

Consejos de estructura narrativa

  • Mantén tu pasaje de inicio simple – establece el tono de toda la experiencia.
  • Usa decisiones para crear opciones significativas, no solo botones de "continuar".
  • Usa @menciones para crear hipervínculos dentro del texto para lectura no lineal.
  • Usa el tablero para detectar pasajes desconectados o cuellos de botella en tu narrativa.
  • Prueba a menudo – es la forma más rápida de experimentar tu historia desde la perspectiva del lector.
🧪 Experimental: El formato .fmm es JSON plano. Puedes editarlo manualmente para reordenar pasajes (el primero es el inicio) o ajustar posiciones con precisión.
